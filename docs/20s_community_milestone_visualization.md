# 20秒社区发展里程碑可视化方案

目标：基于现有社区 3D 地图，把社区志中带有明确年份或时期的信息整理为时间轴事件，并在 20 秒内完成“空间变化 + 关键事件 + 发展脉络”的短片式可视化。

## 一、成片节奏

| 时间段 | 画面重点 | 地图动作 | 信息层级 |
| --- | --- | --- | --- |
| 0.0-2.0s | 建立全景 | 俯视进入社区范围，边界/核心道路淡入 | 社区名称、时间范围 |
| 2.0-4.5s | 起点时期 | 旧貌/早期地标高亮 | 第 1 个里程碑 |
| 4.5-7.0s | 基础设施变化 | 道路、水系、公共设施依次亮起 | 第 2 个里程碑 |
| 7.0-9.5s | 居住/产业发展 | 建筑体块按年代生长或变色 | 第 3 个里程碑 |
| 9.5-12.0s | 公共服务完善 | 学校、卫生、文化、养老等 POI 高亮 | 第 4 个里程碑 |
| 12.0-14.5s | 治理与更新 | 社区中心、改造范围、治理网格显现 | 第 5 个里程碑 |
| 14.5-17.0s | 当代成果 | 当前 3D 地图完整亮起，关键节点连线 | 第 6 个里程碑 |
| 17.0-20.0s | 收束总结 | 镜头拉回，时间轴完整展开 | 一句话总结 |

建议事件数量控制在 5-7 个。20 秒内超过 7 个事件会变成“字幕翻页”，空间表达会被挤掉。

## 二、视觉规则

1. 时间轴放在屏幕底部，保持固定高度，年份节点随播放进度依次点亮。
2. 3D 地图是主角，字幕只保留“年份 + 事件短标题 + 12-20 字说明”。
3. 每个事件必须绑定一个 `place_id`、坐标或空间范围；无法定位的事件只进入旁白/字幕，不驱动地图镜头。
4. 同类事件使用稳定颜色：
   - 建设更新：橙色
   - 公共服务：蓝色
   - 产业经济：绿色
   - 社区治理：紫色
   - 文化记忆：红色
5. 置信度低的历史范围使用半透明轮廓，不做精确建筑生长动画。

## 三、数据接口

事件数据建议放在 `data/timeline/community_milestones.csv`，核心字段如下：

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `event_id` | 事件编号 | 稳定 ID，如 `cm_001` |
| `title` | 短标题 | 8-14 个字最佳 |
| `start_year` | 起始年份 | 可用具体年份 |
| `end_year` | 结束年份 | 单年事件可留空 |
| `period_label` | 时期文本 | 如“20世纪80年代”“改革开放初期” |
| `event_type` | 事件类型 | construction/service/economy/governance/culture |
| `place_id` | 关联地点 | 对应 3D 地图或 `places` 表 |
| `x` / `y` / `z` | 空间坐标 | 由现有 3D 地图坐标系决定 |
| `camera_target` | 镜头目标 | 可填地点名或坐标 |
| `visual_action` | 视觉动作 | highlight/grow/path/reveal/compare |
| `source_text` | 社区志原文摘录 | 保留证据 |
| `source_ref` | 来源位置 | 章节、页码或文件名 |
| `confidence` | 置信度 | high/medium/low |
| `duration_s` | 建议时长 | 2-3 秒 |

## 四、3D 地图联动逻辑

播放控制器只需要维护三个状态：

```text
current_time_s
active_event_id
active_year
```

每个事件触发时：

1. 时间轴节点点亮。
2. 镜头飞向 `camera_target`。
3. 根据 `visual_action` 执行动画：
   - `highlight`：地点发光/描边。
   - `grow`：建筑或范围从地面生长。
   - `path`：道路、河道、线路按方向绘制。
   - `reveal`：图层淡入。
   - `compare`：旧貌图层与现状模型做透明切换。
4. 显示事件标题、年份和一句说明。

## 五、制作流程

1. 从社区志中抽取所有带时间的信息，先不筛选。
2. 合并同年同类事件，删去无法在 20 秒内讲清的细枝末节。
3. 选出 5-7 个能代表发展阶段的里程碑。
4. 为每个事件绑定空间位置：优先用 3D 地图对象 ID，其次用坐标，最后用社区范围中心点。
5. 填写 `community_milestones.csv`。
6. 在现有 3D 地图中接入播放控制器。
7. 录制或导出 20 秒视频。

## 六、需要补齐的输入

当前仓库已有地图、QGIS 和地点数据，但还缺两类关键输入：

1. 现有社区 3D 地图的工程位置或访问地址。
2. 社区志文本、PDF、Word，或人工整理出的事件清单。

拿到这两项后，就可以进入事件抽取和原型接入。
