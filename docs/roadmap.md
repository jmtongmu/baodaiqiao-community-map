# 项目路线

## 阶段 1：POC 固化

目标：把已完成的 QGIS/PyQGIS POC 变成可复用项目资产。

已完成：

- 归档 QGIS 工程、工程依赖、参考图像和 PyQGIS 脚本。
- 从脚本中抽取 73 个地点对象，生成 CSV/GeoJSON。
- 保留每个对象的名称、类别、坐标、置信度和定位逻辑。

下一步：

- 统一 CRS。
- 把 memory layer 固化到 GeoPackage。
- 建立版本字段和证据字段。

## 阶段 2：空间锚点体系

目标：让 AI 的空间判断围绕稳定锚点展开。

建议优先锚点：

- 十三城门
- 城墙线
- 皇城/大内范围
- 御街轴线
- 西湖岸线和钱塘江方向
- 关键桥梁、河道交汇、山体节点

每个锚点建议记录：

| 字段 | 含义 |
| --- | --- |
| `anchor_id` | 锚点编号 |
| `name` | 历史名称 |
| `modern_ref` | 现代参考地物 |
| `x` / `y` | EPSG:3857 坐标 |
| `confidence` | high/medium/low |
| `source` | 地图、志书、现代资料来源 |
| `notes` | 校对说明 |

## 阶段 3：AI 地图理解迭代

目标：从老地图中继续抽取地名、功能区和空间关系。

推荐循环：

```text
选定地图局部
  -> AI 识别文字和地物
  -> 输出地名、类别、相对关系、证据片段
  -> 转成 places/evidence/relations
  -> QGIS 叠图校验
  -> 调整坐标和置信度
```

AI 输出不要直接变成最终坐标，应先成为“空间假设”。坐标生成后还要经过 QGIS 视觉校验和人工确认。

## 阶段 4：2D Web 地图

目标：把 QGIS 中的复原图层输出为可交互地图。

推荐最小产品：

- 底图：OSM/历史栅格切换
- 图层：城墙、皇城、御街、城门、地名面状对象
- 交互：点击对象显示定位逻辑、置信度、来源
- 筛选：类别、置信度、版本

可选输出方式：

- QGIS + qgis2web
- GeoJSON + Leaflet/OpenLayers
- GeoPackage/GeoJSON -> 矢量瓦片 -> Web 地图

## 阶段 5：3D 城市复原

目标：从 2D 空间骨架进入可浏览的历史城市。

进入 3D 前需要区分三类几何：

| 等级 | 含义 | 是否适合直接建模 |
| --- | --- | --- |
| `symbolic` | 当前 POC 的符号面/示意面 | 否 |
| `estimated_area` | 历史推定范围 | 可用于体块 |
| `footprint` | 较可信建筑轮廓 | 可用于精细建模 |

推荐先做低精度体块城市：

- 城墙、城门、皇城作为高优先级模型
- 御街和水系作为空间骨架
- 坊巷和市场用体块/纹理表达
- 低置信度对象只显示为半透明范围

## 阶段 6：志书事件时间轴

目标：把地方志、社区志中的事件绑定到地点和时间。

建议事件字段：

| 字段 | 含义 |
| --- | --- |
| `event_id` | 事件编号 |
| `title` | 事件标题 |
| `start_year` / `end_year` | 时间范围 |
| `dynasty` | 朝代 |
| `place_id` | 关联地点 |
| `people` | 相关人物 |
| `source_text` | 志书原文摘录 |
| `source_ref` | 来源页码或章节 |
| `confidence` | 置信度 |

时间轴不要只做年代列表，最好和地图联动：点击年份可以看到城市空间中发生过什么，点击地点也能看到这个地点跨年代的事件。
